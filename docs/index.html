Core Concepts

Experiment-Specific Reaction Configuration

ePICReaction is the core class for ePIC data analysis, inheriting from ElectroIonReaction in rad. It provides streamlined setup for ePIC event files, particle assignment, and MC matching.

  • Handles both reconstructed and truth (MC) particle information in parallel
  • Supports resolution studies and truth/reco matching (optional, not available for real data)
  • Flexible particle definitions and kinematic calculations

Detector-Level Association

ePICDetectorReaction extends ePICReaction to associate low-level detector objects (tracks, clusters) with particles.

  • Links calorimeter, tracker, and other detector info to reconstructed particles
  • Uses podio metadata for collection IDs and association logic
  • Critical for advanced analyses needing full detector context

MC Matching: When enabled, truth-reco associations allow resolution and efficiency studies. For real data, rely on particle identification and reconstructed info only.

Code Structure & Workflow

Class Hierarchy & Usage Flow
ElectroIonReaction (from rad) ↓ ePICReaction (event-level, ePIC-specific) ↓ ePICDetectorReaction (detector-level, associations) ↓ Analysis Modules ParticleCreator, Histogrammer, Utilities, MC Matching
Note: For MC studies, enable truth-reco matching to create resolution and efficiency columns. For real data, use particle identification and reconstructed info only.
Typical Analysis Workflow
  • Choose interface: Use ePICReaction for event-level, or ePICDetectorReaction for full detector info
  • Configure analysis: Assign particles, set up MC matching if available
  • Associate detector info: Use AssociateTracks() and AssociateClusters() for calorimeter/tracker links
  • Define analysis steps: Kinematics, selection, histogramming, resolution studies
  • Run interactively: With ROOT (e.g. root examples/ProcessMCMatchedY.C)

Detector-Level Analysis (ProcessMCMatched_DVCS.C)

// Setup for advanced detector association
#include "ePICDetectorReaction.h"
#include "ParticleCreator.h"
#include "Indicing.h"
#include "Histogrammer.h"

void ProcessMCMatched_DVCS(){
  using namespace rad::names::data_type;
  rad::config::ePICDetectorReaction epic{"events", "datafile.root"};
  epic.SetBeamsFromMC();
  epic.AliasColumnsAndMatchWithMC();

  // Associate calorimeter info
  epic.AssociateClusters({"EcalBarrelClusters","EcalEndcapNClusters"}, {"energy"});
  // Associate tracker info
  epic.AssociateTracks({"TaggerTrackerTracks"}, {"momentum.x"});
  // Continue analysis...
}
// Run with: root examples/ProcessMCMatched_DVCS.C
